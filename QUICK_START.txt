SECRETS PROTECTION (CRITICAL)
================================================================================

Read SECRETS_MANAGEMENT_GUIDE.md

DO NOT COMMIT:
  ❌ .env (your real API keys)
  ❌ credentials.json (broker token)
  ❌ config.json (your personal settings)
  ❌ logs/ (your trading history)

DO COMMIT:
  ✅ .gitignore (tells git to ignore secrets)
  ✅ .env.example (template for others)
  ✅ Code fixes
  ✅ Tests

How it works:
  - .gitignore tells git to ignore .env, credentials.json, etc.
  - You create .env locally with YOUR credentials
  - Git won't track it
  - Your secrets never touch GitHub

COMMON MISTAKES (AVOID THESE)
================================================================================

❌ Skip writing tests first (write code first)
   → Tests are the proof the fix works
   → Without tests, bugs will return

❌ Commit multiple bugs in one commit
   → One logical change per commit
   → Makes history readable

❌ Ignore code review
   → Review catches 50%+ of bugs
   → Senior dev perspective is valuable

❌ Push credentials to GitHub
   → .gitignore prevents this automatically
   → But double-check before pushing

❌ Try to fix all 21 bugs at once
   → Follow the 4-week plan
   → Do 8 bugs/week in priority order

❌ Change the 4 principles to "go faster"
   → These principles prevent future bugs

4 CORE PRINCIPLES (MUST FOLLOW FOR EVERY WORK):
================================================================================

✅ PRINCIPLE 1: Test-First Development
   Write tests BEFORE code. All work tested with 80%+ coverage.

✅ PRINCIPLE 2: One Commit Per Bug
   Each bug/feature = one clear, focused commit.

✅ PRINCIPLE 3: Code Review Mandatory
   All code reviewed before merge. Issues fixed before merging.

✅ PRINCIPLE 4: Security First
   No hardcoded secrets. No credential leaks. No vulnerabilities.

================================================================================
DOCUMENTATION RULE - NO BLOAT:
================================================================================

✅ UPDATE existing docs (BUG_REGISTRY.md, FIX_LOG.md)
❌ DON'T create new docs unless absolutely essential
❌ DON'T duplicate information
❌ DO archive old docs when no longer relevant

Bloat Check Before Creating Any New Doc:
- Does this info exist elsewhere? → Update that instead
- Is this essential? → NO? Don't document it
- Can this fit in existing docs? → YES? Add there instead
- Have we planned archival? → If not, don't create it

Result: Clean, lean, maintainable documentation.
